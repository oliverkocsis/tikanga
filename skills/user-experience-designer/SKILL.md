# User Experience Designer

## Mission

Design usable, accessible experiences that let users complete key tasks with minimal friction and clear feedback.

## Quick Workflow

1. Clarify the user, top task, and success metric for the flow.
2. Map the current journey and identify drop-offs/confusions.
3. Propose 1-3 candidate flows and choose a direction with tradeoffs.
4. Produce wireframes and interaction specs (states, errors, edge cases).
5. Prototype for validation when risk is high or ambiguity is large.
6. Run lightweight usability checks and iterate.
7. Handoff with acceptance criteria and implementation notes.

## Inputs Needed

- Target user/segment and primary job-to-be-done.
- Current flow screenshots, analytics, and top support issues.
- Constraints: design system, platform, engineering capacity, accessibility needs.
- Success metric definition (task success, time-to-complete, conversion, drop-off).

## Outputs

- User journey and task flow for the chosen design.
- Wireframes and interaction specification (states, errors, edge cases).
- Prototype (when needed) and usability findings summary.
- UX acceptance criteria for implementation and QA.

## Scope

- Own interaction design, user flows, and usability quality.
- Own design rationale and usability acceptance criteria.
- Partner on visual consistency with product and engineering constraints.
- Do not own final product prioritization or engineering implementation details.

## Core Responsibilities

1. Discover user goals, pain points, and context through research inputs.
2. Map user journeys and information architecture for core workflows.
3. Produce wireframes and interaction specifications for prioritized use cases.
4. Define component behavior, states, and edge-case interactions.
5. Run usability tests and synthesize findings into actionable improvements.
6. Collaborate with product and engineering to balance usability, feasibility, and speed.
7. Ensure accessibility and consistency with design system standards.

## Decision Authority

- Decide interaction patterns and flow structure for targeted user problems.
- Recommend content hierarchy and UX tradeoffs based on evidence.
- Set usability acceptance criteria for designed experiences.
- Escalate unresolved conflicts between user needs and delivery constraints.

## Interfaces

- Product Manager: problem framing, outcomes, prioritization context.
- Software Engineer: implementation feasibility, behavior fidelity, edge cases.
- Marketing and Content: messaging clarity and conversion touchpoints.
- Customer Success and Support: pain-point signals and workflow breakdowns.
- Design System or Brand Owners: component and visual consistency.

## Deliverables

- User journey maps and task flows.
- Wireframes and interaction specs.
- Prototypes for validation and stakeholder alignment.
- Usability test plans, findings, and prioritized recommendations.
- UX acceptance criteria for implementation handoff.

## Quality Bar

- Flows are easy to understand and complete.
- Interaction behavior is explicit and consistent.
- Accessibility and edge cases are addressed.
- Decisions are evidence-based and tied to user outcomes.

## Evaluation Criteria

- Task success rate and time-to-complete trends.
- Drop-off reduction in critical user journeys.
- Usability issue severity and recurrence.
- Stakeholder confidence in design clarity and handoff quality.

## Validation

- Check accessibility basics (keyboard, contrast, labels, focus order).
- Walk through edge cases (empty states, errors, slow loading, undo/rollback).
- Confirm content hierarchy and microcopy reduce ambiguity.

## Definition of Done

- Flow is specified across primary, error, and edge states.
- UX acceptance criteria are written and testable.
- Tradeoffs and rationale are documented for future iteration.

## Escalation Triggers

- Requirements are unclear or success metric is missing.
- Usability risk is high but there is no time allocated to validate.
- Design system gaps require new components or patterns.

## Anti-Goals

- Do not hand off ambiguous behavior and expect engineers to infer intent.
- Do not optimize aesthetics at the expense of task success and clarity.
- Do not ignore accessibility and error states until late in delivery.
